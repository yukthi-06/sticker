# Build Specification: Windows Desktop Sticker Application (C# + WinUI 3)

You are a senior Windows desktop software architect and C# developer.

Design and implement a production-quality Windows desktop "Sticker" application using C# (.NET 9+) and WinUI 3 (Windows App SDK).

The application behaves like desktop stickers/notes that are attached to the Windows desktop itself and are never displayed above normal application windows.

---

# Project Goal

Create a lightweight, portable Windows 11 desktop sticker application that allows users to place visual stickers on the desktop.

The stickers should appear as part of the desktop experience, remaining visible when the user presses Win+D (Show Desktop), while automatically staying behind normal application windows when applications are restored.

The application should have a modern Windows 11 appearance, support transparency, animations, drag-and-drop positioning, and store all data in portable JSON files.

# Implementation Plan - Windows Desktop Sticker Application (C# + WinUI 3)

## Technical Approach & Architecture

We will implement a clean, portable WinUI 3 application using the Windows App SDK (.NET 9) with MVVM pattern.

### 1. Desktop WorkerW Integration Technique
To ensure stickers remain visible during Win+D (Show Desktop) and stay behind other applications:
- Find the `Progman` window using `FindWindow`.
- Send message `0x052C` (WM_SPAWN_WORKER) to `Progman`. This prompts Windows to spawn a new `WorkerW` window.
- Enumerate top-level windows using `EnumWindows` to find the `WorkerW` window that hosts the desktop wallpaper and is a sibling of the `WorkerW` containing `SHELLDLL_DefView` (which hosts desktop icons).
- Reparent sticker windows to this `WorkerW` window using `SetParent`.
- For stickers set to "Always On Top", we remove them from the `WorkerW` parent (setting parent back to `IntPtr.Zero` / null) and apply `HWND_TOPMOST` via `SetWindowPos`.
- For click-through, we manipulate window styles using `GetWindowLongPtr` and `SetWindowLongPtr`, applying `WS_EX_TRANSPARENT` and `WS_EX_LAYERED`.

### 2. Project Directory Structure
We will structure the project inside a folder named `StickerApp` as follows:
- `StickerApp.csproj`: The WinUI 3 desktop app project.
- `App.xaml` & `App.xaml.cs`: Application entry point, service locator, and initialization.
- `Models/`: Data models for Stickers and Settings.
- `ViewModels/`: MVVM ViewModels.
- `Views/`: XAML Windows/Views.
- `Services/`: Services for storage, desktop hosting, backup, tray, notification, etc.
- `Interop/`: Win32 PInvoke wrapper.

### 3. File Persistence
- Everything is stored in the application directory.
- `settings.json`: Stores app configuration.
- `stickers.json`: Stores sticker list (with soft-delete support).
- `backups/`: Rolling history of up to 50 backups.
- `logs/`: Simple file logger for errors and events.
