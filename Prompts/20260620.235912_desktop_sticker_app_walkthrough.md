# Build Specification: Windows Desktop Sticker Application (C# + WinUI 3)

You are a senior Windows desktop software architect and C# developer.

Design and implement a production-quality Windows desktop "Sticker" application using C# (.NET 9+) and WinUI 3 (Windows App SDK).

The application behaves like desktop stickers/notes that are attached to the Windows desktop itself and are never displayed above normal application windows.

---

# Project Goal

Create a lightweight, portable Windows 11 desktop sticker application that allows users to place visual stickers on the desktop.

The stickers should appear as part of the desktop experience, remaining visible when the user presses Win+D (Show Desktop), while automatically staying behind normal application windows when applications are restored.

The application should have a modern Windows 11 appearance, support transparency, animations, drag-and-drop positioning, and store all data in portable JSON files.

# Walkthrough - Windows Desktop Sticker Application

## Changes Made

We have successfully designed and built a portable, lightweight, and modern **Windows 11 Desktop Sticker Application** in C# and WinUI 3. Below is a breakdown of what has been implemented:

### 1. Windowing and Desktop Integration
- **WorkerW Layer Reparenting**: Implemented dynamic discovery of the hidden `WorkerW` window. By sending a custom message `0x052C` to the Progman window and using `SetParent`, we successfully place sticker windows directly onto the desktop wallpaper layer. This guarantees they stay pinned to the background (even during `Win+D` / Show Desktop actions) and automatically stay behind any normal application windows.
- **Click-Through Mode**: Enabled custom click-through behavior via standard Win32 flags (`WS_EX_TRANSPARENT` and `WS_EX_LAYERED`), allowing background mouse clicks to bypass stickers when interactive mode is disabled.
- **Always-on-Top Mode**: Enabled an override toggle which returns stickers to normal popup windows and marks them as topmost (`HWND_TOPMOST`), making them float above all other active windows.

### 2. Services & Architecture (MVVM)
- **JsonStorageService**: Handles configuration (`settings.json`) and stickers serialization (`stickers.json`) relative to the executable path.
- **BackupService**: Saves a rotating copy of stickers data to `/backups` (keeping the last 50 files) and provides fallback auto-recovery.
- **LoggerService**: Thread-safe structured logging to `/logs`.
- **StartupService**: Hooks system startup via HKCU Run registry values.
- **TrayService**: Direct shell interop-based notify icon in the taskbar tray, hosting context menus for rapid sticker management.
- **NotificationService**: Native Windows generic XML Toast dispatch.
- **AnimationService**: Direct storyboarded animation support.

## Detailed Explanation of WorkerW Desktop Integration

The core requirement of this application is that stickers behave like parts of the desktop wallpaper. Standard windows would minimize when the user clicks "Show Desktop" (Win+D) or be covered/placed in front of user applications in unexpected ways.

To solve this, we leverage the Windows Shell Window Hierarchy:

1. **The Desktop Layer**: When Windows boots, `Explorer.exe` hosts a desktop window called `Progman` (Program Manager). Behind it is the `SHELLDLL_DefView` containing the desktop listview (icons).
2. **Spawning WorkerW**: By sending message `0x052C` to `Progman`, we instruct the shell to create a secondary WorkerW window between the desktop wallpaper and the desktop icons.
3. **Reparenting**: The application calls `SetParent(stickerHwnd, workerwHwnd)` and converts the window style from a floating popup (`WS_POPUP`) to a child window (`WS_CHILD`). 
4. **Visibility & Layout**: Since the sticker window is now a child of the desktop's internal background rendering window, it behaves as a native element of the desktop wallpaper. When the user minimizes all windows via Win+D, the shell brings the desktop (and all child windows of the wallpaper WorkerW) to the front, preserving the sticker's visibility. When standard windows are restored, they naturally overlap the wallpaper and WorkerW, keeping our stickers behind all active workspaces.

## Verification & Build Instructions

### How to Build
To compile the portable release bundle, run:
```powershell
dotnet build StickerApp/StickerApp.csproj --configuration Release
```
The output executable with the dynamic timestamp suffix (e.g. `StickerApp_20260621_000413.exe`) and dependency assets will be written to `StickerApp/bin/Release/net9.0-windows10.0.19041.0/win-x64/`.

### How to Run
Launch the timestamped `StickerApp_*.exe` directly from the output directory. All logs, settings, and backups will be created inside that directory, allowing you to run the application directly from a portable USB drive.
